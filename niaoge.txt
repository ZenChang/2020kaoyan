第零章:计算器概论

--0.1 计算机的概念:辅助人脑的工具

计算机其实就是计算器:
接收用户的指令与数据,经由中央处理器的算术和逻辑单元运算处理后,以产生或存储成有用的信息

输入设备:键盘鼠标
输出设备:显示器打印机
所以让一个机器有个输入然后有个输出,那么它就是个计算机了

0.1.1 计算机的组成:五大单元
输入单元:键盘鼠标扫描仪手写板触控屏卡片阅读器
主机部分:系统单元:主机外壳包裹,主板 + (cpu + 主存储器 + 硬盘(外部存储装置)), 对于手机来说是焊接在一起的.
输出单元:显示器打印机

cpu:central processing unit 含有微指令集的芯片 = 算逻单元 + 控制单元
算逻单元:运算和逻辑判断
控制单元:协调组件和算逻单元的工作

流程:输入单元-->主存储器-->CPU-->主存储器-->输出单元

计算机组成部分:输入单元 + 算逻单元 + 控制单元 + 主存储器 + 输出单元

0.1.2 cpu架构:一切设计的起点

cpu架构:RISC || CISC

RISC:ARM系列
CISC:x86系列 指令数目多 指令长度不同 指令时间较长

0.1.3 其他单元

适配器
辅存:硬盘软盘光盘磁带

0.1.5 计算机用途分类
超级计算机 大型计算机 迷你计算机 工作站 微计算机

0.1.6 计算机上常用的计算单位(容量 速度)

单位:kilo,mega,giga,tera,peta,exa,zetta

容量一般是二进制单位
速度一般是十进制单位

--0.2 个人计算机架构与相关组件
主板:北桥(连接快速) 南桥(连接慢速)

北桥由于连接CPU和主存储器 因此常常被封装在CPU内部了.所以主板上只能看到南桥

0.2.1 cpu

双核cpu:一个cpu外壳里面有连个算术单元
cpu频率:cpu每秒钟进行工作的次数,工作的含义是进行少数的指令操作
频率目前只能同款cpu进行比较,因为不同款的cpu的每个指令的长度,

cpu外频:cpu与外部组件进行数据传输时的频率
cpu倍频:cpu内部加速的倍数
两者相乘才是cpu的频率速度

cpu-z
北桥被抛弃,cpu直接与主存储器和显示器交流

32位与64位的cpu -- 总线宽度
cpu与主存储器的传输速度:FSB 前端总线速度 = 主存储器频率 * 宽度 = 带宽
cpu处理主存储器字节的能力:32位=4GB内存
cpu等级:x86_64等级
超线程:在cpu内部将重要的寄存器分为两部分,让两个程序同时竞争两个算术单元,
好像同时拥有了两个核心,而非操作系统的任务切换

0.2.2 内存
主存储器的主要组件是动态随机存取内存,挥发性内存
双通道:将两个主存储器连接在一起

DRAM和SRAM
cpu(含寄存器)-->cpu(第二层高速缓存)-->系统区流排-->主存储器-->IO区流排-->外部存储装置
L2 cache 和 cpu速度一致:静态随机存取内存

只读存储器
BIOS(basic input output system)是一套程序,该程序是写死到主板上的一个内存芯片中,
这个内存芯片在没有通电时也能够将数据记录下来,那就是只读存储器

firmware就是绑定在hardware上的控制software

0.2.3 显示适配器

VGA = video graphics array
内存容量影响图像的显示和颜色深度
在显示适配器中 + 3D加速芯片 = GPU
显示适配器与显示器之间的连接:1.VGA(较早) 2.DVI(液晶) 3.HDMI(图像和声音) 4.DP(图像和声音)

0.2.4 硬盘和存储设备
硬盘的物理构成:许许多多的圆形磁盘盘 + 机械手臂 + 磁盘读取头 + 主轴马达
主轴马达让磁盘盘转动,然后机械手臂可伸展让读取头在磁盘盘上进行读写

磁盘盘上的数据:
扇区:一个个的小区块,磁盘的最小物理存储单位
磁道:同一个同心圆的扇区组成的圆环
磁柱:所有磁盘盘的同一个磁道成为磁柱

传输界面:SATA USB SAS
HDD-->SSD

0.4.2 操作系统
一组程序:管理系统中的所有活动,并驱动系统中的所有硬件
让cpu进行算逻运算,让主存储器加载程序代码,让硬盘可以被存取,让网络卡可以传输数据

操作系统就是接收软件的需求去驱动硬件的核心程序
放置核心程序的区块是受保护的,并且开机后就一直常驻在内存中.
系统呼叫:核心的系统呼叫接口可以将C语言的相关语法转换为核心可以了解的任务函数

操作系统的核心是直接参考硬件规格写的
操作系统只是在管理整个硬件功能
应用程序的开发只是在参考操作系统提供的开发接口

核心功能:系统呼叫接口,程序管理,内存管理,文件系统管理,装置的驱动

操作系统与驱动程序:操作系统提供开发驱动的接口,硬件厂商依据接口开发自己的驱动,用户安装该驱动.
驱动程序是硬件厂商提供的,与操作系统开发者无关.

2.2.3 开机流程中的BIOS和UEFI开机检测程序
CMOS:记录硬件参数嵌入主板的存储器
BIOS:写入到主板上的一个韧体(firmware, 写入到硬件上的软件程序),计算机开机执行的第一个程序

BIOS(读取第一个扇区的MBR位置)-->MBR(load核心文件)-->操作系统执行

1. 硬件开机,启动开机主动执行的韧体:BIOS
2. BIOS读取一个扇区的装置,也就是MBR
3. MBR内含开机管理程序(boot loader)
4. boot loader读取操作系统核心文件

boot loader: 提供选单,载入核心文件,转交其他loader

1. 每个分区槽都有自己的启动扇区
2. 实际可开机核心文件是放在各个分区槽中的
3. loader只会认识自己分区槽中的核心文件以及其他loader
4. loader可直接指向其他loader
